如何搭建我的 GitHub Pages 个人网站

这篇文章记录当前个人网站从想法到上线的第一版过程。

目标

我希望这个网站先承担两个职责：第一，作为公开的自我介绍入口；第二，作为技术博客和思考记录的目录。

仓库命名

GitHub 用户主页需要使用 WangAo-0.github.io 作为仓库名。这样部署完成后，公网地址就是 https://WangAo-0.github.io/。

第一版结构

  1. 首页：介绍我是谁、这个网站记录什么。
  2. 技术博客：放搭建记录、项目复盘和技术笔记。
  3. 思考札记：放还没有完全展开，但值得持续追问的问题。
  4. 联系入口：保留 GitHub 和邮箱。

发布方式

当前站点是纯静态文件，不需要构建工具。把文件提交到 main 分支后，在 GitHub Pages 中选择从 / (root) 发布即可。

后续迭代

后续我会逐步补充真实文章，并把每次项目实践中的关键决策、问题排查和复盘写下来。这样网站就不是静态名片，而是持续增长的知识库。
