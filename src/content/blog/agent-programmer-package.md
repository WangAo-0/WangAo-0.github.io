## 核心理解

我理解的 agent 程序员包，不是简单让模型帮我写一段代码，而是把一个 agent 需要的外围环境和开发准则打包好。

## 它应该覆盖什么

从最开始的沟通，到产品设计、开发、测试、上线、维护、优化和新需求增加，都应该有一套可执行流程。用户输入应该尽量集中在目标和约束上，输出则是可以直接使用的完整产品。

## 关键难点

真正难的不是生成代码，而是让 agent 知道什么叫“完成”：功能是否满足需求，测试是否通过，体验是否可用，后续维护是否容易。
